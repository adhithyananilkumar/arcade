# AI Agent Rules

This document is **mandatory** and defines strict rules for any AI coding agent contributing to the repository.

## Strict Rules
- **Never change project architecture without approval.**
- **Never rename folders unless requested.**
- **Never introduce new libraries without justification.**
- **Prefer existing utilities before creating new ones.**
- **Keep code simple and readable.**
- **Avoid duplicate implementations.**
- **Respect the documented folder structure.**
- **Follow naming conventions.**
- **Do not generate placeholder or mock code unless explicitly requested.**
- **Do not modify unrelated files.**
- **Keep commits focused on a single feature or bug.**
- **Update documentation whenever architectural changes are introduced.**

## Agent Audit Checklist
Every AI-generated contribution MUST be verified against this checklist:
- [ ] Folder structure followed
- [ ] Naming conventions followed
- [ ] No duplicated code introduced
- [ ] No unused files created
- [ ] No unnecessary dependencies added
- [ ] TypeScript errors checked
- [ ] Lint issues resolved
- [ ] Documentation updated (if required)
- [ ] Existing architecture respected
- [ ] Scope limited to the requested task
